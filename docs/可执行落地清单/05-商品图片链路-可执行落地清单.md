# 05-商品图片链路-可执行落地清单

## A. 一次性初始化

1. 确认目录存在：
   - `backend/storage/goods-img/`
   - `backend/storage/upload/`
2. 确认后端配置已生效：
   - `/goods-img/**` 映射到 `./storage/goods-img/`
   - `/upload/**` 映射到 `./storage/upload/`

## B. 数据和图片的对应规则

1. 数据库字段：`beyeah_mall_goods_info.goods_cover_img`
2. 对应规则：字段值是 Web 路径，不是文件系统路径。
3. 推荐写法：`/goods-img/<文件名>`
4. 实际文件放置：`backend/storage/goods-img/<文件名>`
5. 结论：默认按“文件名”对应，不按“商品ID”自动对应。

## C. 你实际要做的最短流程

1. 把商品图复制到 `backend/storage/goods-img/`。
2. 新增/修改商品时，把 `goodsCoverImg` 填成 `/goods-img/文件名`。
3. 启动后端和前端。
4. 刷新页面确认显示。

## D. 快速自检命令

1. 运行缺图扫描脚本（仓库根目录）：

```powershell
.\scripts\assets\check-goods-img.ps1
```

2. 查看缺失清单：
   - `docs/goods-img-missing.txt`
3. 随机抽查一张图的 URL：

```text
http://localhost:23333/goods-img/<文件名>
```

## E. 如果你想按商品ID命名图片

说明：系统默认不做商品ID自动映射。要按商品ID命名，必须更新数据库路径。

示例（商品ID=10003，文件名=10003.jpg）：

```sql
UPDATE beyeah_mall_goods_info
SET goods_cover_img = '/goods-img/10003.jpg',
    goods_carousel = '/goods-img/10003.jpg'
WHERE goods_id = 10003;
```

## F. 前端容错验收

1. 人为把某商品 `goods_cover_img` 改成不存在文件路径。
2. 访问首页/分类/详情/购物车/结算。
3. 预期：显示占位图 `goods-placeholder.svg`，不出现破图。

## G. 回滚方案

1. 若图片链路异常，优先回滚到“数据库路径 + 文件名一致”策略。
2. 保证至少一张可访问图片后，再批量导入其余图片。
3. 上传链路异常时，先验证 `backend/storage/upload/` 写权限。

## H. 使用 dummyjson 一键补图（推荐）

1. 运行脚本（仓库根目录）：

```powershell
.\scripts\assets\fill-goods-images-from-dummyjson.ps1 -DownloadCount 100
```

2. 脚本产物：
   - 下载图片到：`backend/storage/goods-img/dummyjson-*.webp`
   - 生成 SQL：`database/migrations/V20260308_05_fill_goods_images_from_dummyjson.sql`

3. 将生成 SQL 导入你的 MySQL（`beyeah` 库）。
4. 重启后端并刷新前端页面。
