# 第一阶段：补全残缺、统一契约与接口（大刀阔斧版）

## 0. 阶段目标
在不大改前端页面展示逻辑的前提下，重建唯一可用交易主链路：
- 首页商品（新品/热销）
- 分类、搜索、详情
- 购物车增删改查
- 提交订单、订单列表、订单查询
- 登录注册、用户信息

完成标准：当前 Vue 页面可真实驱动数据库数据变化，不再出现“前端显示成功、后端无效”。

## 1. 契约选择（最终定版）
### 1.1 新 API 前缀
统一为：`/api/v1`

### 1.2 新接口清单（必须落地）
1. 首页  
- `GET /api/v1/home/new`  
- `GET /api/v1/home/hot`

2. 商品  
- `GET /api/v1/goods?categoryId=&keyword=&sortField=&page=&pageSize=`  
- `GET /api/v1/goods/{goodsId}`

3. 购物车  
- `GET /api/v1/cart`  
- `POST /api/v1/cart/items`  
- `PUT /api/v1/cart/items/{cartItemId}`  
- `DELETE /api/v1/cart/items/{cartItemId}`

4. 订单  
- `POST /api/v1/orders`  
- `GET /api/v1/orders?page=&pageSize=`  
- `GET /api/v1/orders/{orderNo}`

5. 用户与认证  
- `POST /api/v1/auth/register`  
- `POST /api/v1/auth/login`  
- `GET /api/v1/users/me`  
- `PUT /api/v1/users/me`

3. 商品详情可打开、加购后购物车有真实记录。  
4. 购物车删除后数据库 `beyeah_mall_shopping_cart_item` 变化正确。

6. `GET /api/v1/orders` 可返回当前用户订单。

## 6. 风险与兜底
1. 风险：现有 session 路径与新 token 路径并存导致混乱。  
- 兜底：第一阶段直接禁用旧 mall controller 路由，不双跑。

2. 风险：前端页面默认字段与后端 VO 不一致。  
- 兜底：在 `service` 层加字段适配函数，页面不改结构。

3. 风险：旧数据编码混乱导致脏文本。  
- 兜底：接口输出前统一字符清洗，后续第二阶段做彻底修复。
