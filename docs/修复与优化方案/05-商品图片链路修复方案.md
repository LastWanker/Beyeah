# 05-商品图片链路修复方案

## 1. 目标
解决当前前端“商品无图/破图”问题，达到学生作品可用标准：

1. 后端有固定相对目录可放图。
2. 数据库图片路径和文件目录能稳定对应。
3. 前端即使图片缺失也有占位图，不再空白。

## 2. 现状观察（已核对）

1. 数据库 `goods_cover_img` 主要是 `/goods-img/<文件名>` 形式，少量历史数据是 `http://localhost:28089/goods-img/...`。
2. 项目原本没有有效的商品图片静态目录映射，且上传目录是硬编码绝对路径 `D:\upload\`。
3. 前端直接渲染 `goodsCoverImg`，没有统一 URL 归一化与失败兜底。

结论：问题是“图片链路不完整”，不是单一页面组件问题。

## 3. 图片对应关系设计（核心）

### 3.1 采用“文件名路径映射”，不是“商品ID硬绑定”

当前系统的真实绑定键是：`goods_cover_img` 字段里的路径字符串。

示例：

- 数据库存：`/goods-img/87446ec4-e534-4b49-9f7d-9bea34665284.jpg`
- 文件应放：`backend/storage/goods-img/87446ec4-e534-4b49-9f7d-9bea34665284.jpg`
- 前端访问：`http://<backend-host>/goods-img/87446ec4-e534-4b49-9f7d-9bea34665284.jpg`

所以默认策略是“按文件名对应”。

### 3.2 商品ID策略作为可选辅助

如果希望按商品ID命名（例如 `10003.jpg`），必须同步更新数据库 `goods_cover_img` 为 `/goods-img/10003.jpg`，否则系统无法自动知道这层映射。

## 4. 本次落地改动

## 4.1 后端目录与映射

1. 新增相对目录：
   - `backend/storage/goods-img/`
   - `backend/storage/upload/`
2. 后端增加静态映射：
   - `/goods-img/** -> ./storage/goods-img/`
   - `/upload/** -> ./storage/upload/`
3. 上传目录常量从绝对路径改为相对路径。
4. 上传目录创建逻辑从 `mkdir` 改为 `mkdirs`，支持多级目录自动创建。

## 4.2 前端容错与兼容

1. 新增图片工具：`frontend/src/utils/image.js`
   - 统一将相对路径补成当前后端地址。
   - 自动兼容历史 `localhost:28089/goods-img/...` 链接。
2. 新增占位图：`frontend/public/images/goods-placeholder.svg`
3. 在商品卡片、商品详情、购物车、结算页统一加加载失败兜底。

## 4.3 辅助排查脚本

新增脚本：`scripts/assets/check-goods-img.ps1`

用途：
1. 从 `database/InsertData.sql` 提取所有 `/goods-img/文件名`。
2. 对比 `backend/storage/goods-img/` 现有文件。
3. 输出缺失清单到 `docs/goods-img-missing.txt`。

## 5. 学生作品建议用法（最省事）

1. 把图片直接扔到 `backend/storage/goods-img/`。
2. `goods_cover_img` 存成 `/goods-img/你的文件名.jpg`。
3. 重启后端，刷新前端即可看到。
4. 文件缺失时前端会显示占位图，不会再白块。

## 6. 验收标准

1. `http://localhost:23333/goods-img/<文件名>` 能直接访问图片。
2. 首页、分类页、详情页、购物车、结算页均不出现破图图标。
3. 人为填一个不存在的图片路径时，页面显示占位图。
4. 脚本能输出缺失文件清单，便于补图。

## 7. 外部 API 自动填充方案（dummyjson）

可用，而且已落地为本项目脚本化能力。

设计原则：

1. 不直接依赖外链展示，先下载到本地 `backend/storage/goods-img/`。
2. 再批量更新数据库为相对路径 `/goods-img/xxx.webp`。
3. 保持你项目“相对路径静态托管”的一致性。

已提供：

1. 脚本：`scripts/assets/fill-goods-images-from-dummyjson.ps1`
2. 生成迁移 SQL：`database/migrations/V20260308_05_fill_goods_images_from_dummyjson.sql`

脚本完成的动作：

1. 从 `https://dummyjson.com/products` 拉取商品缩略图 URL。
2. 下载图片到 `backend/storage/goods-img/`（命名为 `dummyjson-001.webp` 等）。
3. 解析 `database/InsertData.sql` 的商品ID。
4. 生成批量 `UPDATE` SQL（同时更新 `goods_cover_img` 和 `goods_carousel`）。
