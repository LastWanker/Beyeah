# 第二阶段：稳定性、质量与工程化

## 0. 阶段目标
把“能跑”升级为“可靠可维护”，防止再次出现“复活成功但业务空心”。

## 1. 架构收敛与依赖治理
### 1.1 依赖收敛
1. 清理不再使用的依赖  
- `backend/pom.xml:47` `mybatis-plus-boot-starter`（若阶段一已清理相关代码则可移除）  
- `backend/pom.xml:30` thymeleaf 已注释，继续保持后端 API-only（不恢复模板模式）

2. JDK 与构建统一  
- 当前：`backend/pom.xml:22` 为 `java.version=1.8`  
- 目标：统一到 JDK 17（兼容 Spring Boot 2.7.5）并在 README 固化。

3. 配置外置化  
- 当前明文：`backend/src/main/resources/application.properties:20-21`、`backend/src/main/java/SCG/beyeah1211/util/DatabaseUtil.java:17-18`  
- 目标：移除明文账号密码，改环境变量 + `.env.example`。

### 1.2 代码风格统一
1. 数据访问只保留 MyBatis 路径  
- 移除业务层 JDBC/JdbcTemplate 直连：`PersonalController.java:125,165,179`、`MallUser.java:35-107`

2. 统一异常体系  
- 当前：`beyeahExceptionHandler` 直接 `e.printStackTrace()`：`backend/src/main/java/SCG/beyeah1211/controller/common/beyeahExceptionHandler.java:30`  
- 目标：统一错误码与结构化日志，禁止 `printStackTrace`。

3. 统一返回结构  
- 清除双 `Result` 类并存问题：`util/Result.java` 与 `dao/Result.java`。

## 2. 鉴权与安全
## 2.1 API 鉴权
1. 增加认证中间件（拦截器或过滤器）  
- 未登录访问 `/api/v1/cart`、`/api/v1/orders`、`/api/v1/users/me` 必须返回 401。

2. 前端路由守卫  
- 当前 `frontend/src/router/index.js` 无 `beforeEach`：`frontend/src/router/index.js:11-47`  
- 增加登录态守卫，保护 `/cart`、`/checkout`、`/info`。

3. Axios 凭据与 token 注入  
- 当前 `frontend/src/utils/axios.js:3-17` 只有基础拦截。  
- 增加 token 头注入、401 自动处理、统一错误提示。

## 2.2 安全基线
1. 密码策略  
- 注册最小长度、复杂度策略；避免弱口令。  
2. 登录防爆破  
- 增加失败次数/时间窗限制（IP+账号维度）。  
3. SQL 与输入清洗  
- 所有模糊查询使用参数化 + 长度限制。  
4. CORS 统一配置  
- 替代分散 `@CrossOrigin`，改全局白名单配置。

## 3. 测试与可观测性
## 3.1 自动化测试（最小集）
1. 后端集成测试（建议 Testcontainers + MySQL 8）  
- 登录注册  
- 商品查询  
- 购物车增删改  
- 下单成功与失败路径  
- 订单查询

2. 前端关键流程 E2E（建议 Playwright）  
- 登录 -> 搜索 -> 详情 -> 加购 -> 下单

## 3.2 观测与排障
1. 结构化日志（JSON）+ traceId 透传。  
2. 核心业务日志点  
- 登录失败  
- 下单失败原因  
- 库存不足  
- 支付状态变更  
3. 慢 SQL 监控与索引命中检查。

## 4. 数据质量与编码治理
1. 统一项目编码为 UTF-8（源码、资源、数据库连接）。  
2. 清理乱码注释与错误提示（优先 controller/service 层）。  
3. 逐步替换历史拼写不一致字段命名（`username/getusername` 一类）。

## 5. 阶段验收标准
1. 无明文数据库密码出现在仓库。  
2. 关键接口有自动化测试并纳入 CI。  
3. 交易主链路错误可通过日志定位到具体请求。  
4. 未登录访问受保护资源统一返回 401。  
5. 代码库中不再存在业务路径 `printStackTrace`。
