# 接口错误码与排障手册

## 返回结构
```json
{
  "resultCode": 200,
  "message": "SUCCESS",
  "data": {}
}
```

## 常见错误码
- `200`：成功
- `400`：业务参数异常（`beyeahException`）
- `401`：未认证或 token 无效
- `429`：登录失败次数超限，触发短时锁定
- `500`：服务端异常

## 关键排障点
1. 先看响应头 `X-Trace-Id`，用于串联服务日志。
2. 若是 `401`：
   - 检查请求头是否带 `Authorization: Bearer <token>`
   - 检查 token 是否过期
   - 检查 `APP_AUTH_TOKEN_SECRET` 是否与签发时一致
3. 若是 `429`：
   - 等待锁定窗口结束（默认 15 分钟）
   - 或更换账号/IP 验证
4. 若是 `500`：
   - 按 `X-Trace-Id` 在后端日志搜索
   - 重点看 controller/service/mapper 报错栈
