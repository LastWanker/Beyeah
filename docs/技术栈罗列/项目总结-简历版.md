# Beyeah 项目总结（简历版）

更新时间：2026-03-08

## 1. 项目定位

Beyeah 是一个前后端分离的电商系统改造项目，围绕“交易闭环可用 + 管理后台可控 + 关键链路安全可靠”进行分阶段重构与增强，覆盖 C 端交易链路、管理员 RBAC、售后退款、地址簿、图片链路、自动化测试与 CI。

---

## 2. 技术栈

### 2.1 后端

- Java 17
- Spring Boot 2.7.5（Web、Session、定时任务）
- MyBatis（XML Mapper）
- MySQL 8.x
- Maven（`mvnw`）
- Lombok

### 2.2 前端

- Vue 3 + Vite 4
- Vue Router 4
- Pinia + pinia-plugin-persistedstate
- Element Plus + Sass
- Axios

### 2.3 测试与工程化

- JUnit 5
- Testcontainers（MySQL 容器化集成测试）
- Playwright（前端 E2E）
- GitHub Actions CI
- PowerShell 一键测试脚本（`scripts/tests/run-all.ps1`）

### 2.4 数据与运维

- 数据库迁移脚本：`database/migrations`（8 个脚本，含回滚脚本）
- 环境变量配置外置（`.env.example` + `application.properties`）
- TraceId 透传（响应头 `X-Trace-Id`）

---

## 3. 架构与设计思路

1. 统一契约：将历史多套接口契约统一到 `/api/v1/**`，避免前后端错位。
2. API-Only 收敛：后端以 REST JSON 为主，剥离 legacy 页面路由依赖。
3. 分层治理：Controller -> Service -> Mapper，核心交易链路统一 MyBatis 访问风格。
4. 渐进式演进：按“可用闭环 -> 稳定性工程化 -> 业务能力增强 -> 管理员专项”四阶段推进。
5. 风险前置：优先处理幂等、防超卖、鉴权、权限、审计日志等高风险点。
6. 可验证交付：每阶段配套自动化测试与 CI 门禁。

---

## 4. 核心实现（已落地）

### 4.1 交易主链路重建（C 端）

- 完整打通商品浏览、购物车、下单、支付（mock）、订单查询。
- 提供认证与用户能力：注册、登录、用户信息查询/更新。
- API 控制器映射约 40 个（`controller/api` 统计）。

### 4.2 订单可靠性增强

- 幂等下单：
  - 支持 `Idempotency-Key` 请求头与请求体幂等键。
  - 数据库唯一索引 `(user_id, idempotency_key)` 防重复落单。
  - 重复请求直接返回已创建订单号。
- 防超卖：
  - 扣减库存采用条件更新（`stock_num >= goods_count`）。
  - 下单链路事务化（扣库存 -> 写订单 -> 写订单项 -> 删购物车）。
- 超时关单：
  - 定时任务扫描超时未支付订单并自动关闭。
  - 关闭后自动回补库存，避免库存长期占用。

### 4.3 地址簿与订单快照

- 新增地址簿 CRUD 与默认地址设置。
- 下单支持 `addressId`，订单写入收货信息快照。
- 后续修改地址不影响历史订单快照，保证业务可追溯。

### 4.4 售后退款链路

- 新增售后申请、列表、详情与管理审核接口。
- 设计售后/退款/订单退款聚合三套状态。
- 审核通过后触发退款流水闭环，重复审核具备幂等处理。

### 4.5 管理后台与 RBAC

- 管理员登录独立于 C 端用户登录（独立 token）。
- `/api/v1/admin/**` 全量接入管理员鉴权拦截。
- 基于 `@RequirePermission` 注解做权限校验（401/403 分离）。
- 数据库补齐 RBAC 五表（角色、权限、用户角色、角色权限、操作审计）。
- 管理写操作落审计日志（操作者、动作、资源、traceId、IP、响应码）。
- 前端新增管理员登录页与控制台（商品/订单/用户/售后四模块）。

### 4.6 图片链路与前端容错

- 静态资源映射：
  - `/goods-img/** -> backend/storage/goods-img/`
  - `/upload/** -> backend/storage/upload/`
- 前端新增图片 URL 归一化与破图占位兜底。
- 提供缺图扫描和批量补图脚本。

## 5. 安全性设计（简历重点）

1. Token 认证：自定义 HMAC-SHA256 签名 token，含过期时间校验。
2. 双令牌体系：用户 token 与管理员 token 分离，避免权限域污染。
3. 访问控制：
   - 用户受保护路由统一拦截（购物车/订单/用户/地址/售后）。
   - 管理路由统一鉴权 + 细粒度 RBAC 权限校验。
4. 防暴力破解：登录失败按“IP + 账号”维度限次锁定（429）。
5. 输入校验与清洗：
   - 用户名、密码、地址、售后字段长度与格式校验。
   - 文本基础转义（如 `<`、`>`）降低注入风险。
6. 业务一致性安全：
   - 幂等键 + 唯一索引防重复下单/重复退款。
   - 事务与条件更新防并发超卖。
7. CORS 安全：
   - 全局白名单配置，支持 localhost/127.0.0.1 回环别名。
8. 可追踪审计：
   - TraceId 全链路透传，便于故障定位。
   - 管理写操作审计日志落库，支持事后追溯。
9. 密钥与配置管理：
   - 数据库与鉴权密钥环境变量外置，不在代码中硬编码生产值。

## 6. 工程化与质量保障

1. 后端集成测试：
   - Testcontainers + MySQL，覆盖认证、购物车、下单、幂等、地址簿、售后、RBAC、审计等主路径。
   - `ApiIntegrationTest` 约 11 个测试用例。
2. 前端 E2E：
   - Playwright 覆盖 C 端交易流与管理员控制台 smoke。
   - 对关键接口进行稳定 mock，校验鉴权头与关键交互。
3. CI 门禁：
   - GitHub Actions 并行执行后端测试、前端 E2E、前端构建。
4. 本地一键回归：
   - `scripts/tests/run-all.ps1` 串联后端测试 + E2E + build。

## 7. 项目亮点（可直接放简历）

1. 主导老系统“多契约错位”治理，统一为 `/api/v1` 并重建电商交易闭环。
2. 在订单链路落地“幂等 + 事务 + 条件扣库存 + 超时关单回补”，显著提升高并发下数据一致性。
3. 从零补齐地址簿、售后退款与管理员 RBAC，形成 C 端到管理端的完整业务闭环。
4. 建立“后端集成测试 + 前端 E2E + CI”质量体系，将回归验证从人工切换为自动化门禁。
5. 完成 TraceId、审计日志、登录限流、权限拒绝（401/403）等安全与可观测性基线建设。

## 8. 简历精简描述模板

### 模板 A（偏后端）

负责 Beyeah 电商系统重构，基于 Spring Boot + MyBatis + MySQL 统一 `/api/v1` 契约并重建交易闭环；落地订单幂等、防超卖、超时关单回补、地址簿快照与售后退款状态机；搭建管理员 RBAC（鉴权拦截 + 权限注解 + 审计日志），并通过 Testcontainers 集成测试、Playwright E2E 与 GitHub Actions CI 建立自动化质量门禁。

### 模板 B（偏全栈）

主导 Vue3 + Spring Boot 电商项目全栈改造：前端完成 C 端与管理员控制台双端能力，后端完成 40+ REST API、双 token 鉴权、RBAC 权限体系与操作审计；重点实现订单幂等、防超卖、售后退款闭环与图片链路容错，并建设“集成测试 + E2E + CI”工程化体系，显著提升系统稳定性与可维护性。
